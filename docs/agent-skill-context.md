# Army Claw Agent Skill 컨텍스트 주입

## 목적

업로드된 skill을 단순 보관하지 않고, 실제 작업 계획 프롬프트에 반영하기 위한 기능이다.

이번 단계는 fine-tuning이나 모델 재학습이 아니다. 활성화된 `SKILL.md` 내용을 작업 계획 컨텍스트로 조합해, LLM 또는 작업 계획 엔진이 해당 절차와 규칙을 참고할 수 있게 만드는 단계다.

## 구현 범위

- 활성화된 skill만 작업 컨텍스트에 포함한다.
- 비활성화된 skill은 제외한다.
- 각 skill의 `SKILL.md` 내용을 읽어 컨텍스트 블록으로 만든다.
- 컨텍스트 길이는 skill별/전체 기준으로 제한한다.
- `/api/agent/plan`에서 사용자 작업 요청과 활성 skill 컨텍스트를 합친 프롬프트를 반환한다.
- React UI에 `Skill 적용 작업 계획` 미리보기 패널을 추가한다.

## API

### `POST /api/agent/plan`

요청 예시:

```json
{
  "task": "월간 보고서를 만들어줘",
  "execute": false
}
```

응답 주요 항목:

- `task`: 사용자 작업 요청.
- `executed`: 현재는 항상 `false`.
- `prompt`: 활성 skill이 주입된 작업 계획용 프롬프트.
- `used_skills`: 프롬프트에 포함된 skill 목록.
- `message`: 현재 단계 설명.

## 현재 한계

- 아직 실제 LLM 호출은 하지 않는다.
- 아직 파일 생성, 명령 실행, 한컴오피스 조작을 수행하지 않는다.
- 현재 단계는 “Skill이 작업 계획에 제대로 반영되는지”를 검증하기 위한 미리보기다.

## 다음 단계

- 생성된 프롬프트를 현재 선택된 LLM Provider에 전달한다.
- LLM이 반환한 계획을 UI에 표시한다.
- 사용자가 승인하면 계획에 따라 도구 실행 단계로 넘어간다.
- 작업 로그에 적용된 skill 목록과 결과를 기록한다.
