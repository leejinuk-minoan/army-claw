# HWPX 양식 제작 가이드

## 기본 원칙

사용자 양식 기반 작성은 원본 HWPX를 직접 수정하지 않는다. Army Claw는 원본을 작업용 복사본처럼 읽고, 플레이스홀더가 치환된 새 HWPX 파일을 별도 경로에 저장한다.

## 권장 플레이스홀더

다음 형식의 대문자 플레이스홀더를 사용한다.

```text
{{DOCUMENT_TITLE}}
{{AUTHOR}}
{{DEPARTMENT}}
{{DATE}}
{{SUMMARY}}
{{BACKGROUND}}
{{RESULT}}
{{CONCLUSION}}
{{SECTION_1}}
{{RESULT_TABLE}}
```

## 작성 방법

- 한글 2024에서 새 HWPX 양식을 만든다.
- 사용자가 나중에 채우고 싶은 위치에 `{{FIELD_NAME}}` 형식의 값을 입력한다.
- 표 셀 안에도 같은 형식의 플레이스홀더를 둘 수 있다.
- 로고, 결재란, 머리말, 꼬리말, 페이지 여백 등 보존해야 하는 요소는 양식에 미리 배치한다.
- 실제 군 문서, 기관 로고, 개인정보가 들어간 양식은 저장소에 커밋하지 않는다.

## 현재 지원 범위

- 본문 section XML의 명시적 플레이스홀더 검색
- 플레이스홀더 치환
- 원본 파일 불변 확인
- BinData 이미지 해시 보존 확인
- HWPX 필수 엔트리와 ZIP 보안 검사

## 아직 제한되는 범위

- 한글 책갈피와 필드의 정밀 분석은 다음 구현 범위다.
- 실제 HWPX 표 객체의 행/열 구조를 생성하는 기능은 아직 제한적이다.
- 자동 목차 필드는 만들지 않고 정적 목차 텍스트를 생성한다.
- 사용자가 승인하지 않은 휴리스틱 입력 후보는 자동 적용하지 않는다.
