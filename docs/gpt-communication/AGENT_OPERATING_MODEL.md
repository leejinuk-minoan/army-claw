# Army Claw 에이전트 운영 모델

작성일: 2026-07-02

## 1. 채팅별 역할

### A. 마스터 에이전트 채팅

이 채팅이 담당한다.

- 전체 8단계 총괄계획 관리
- 현재 단계와 세부 단계 판정
- 제품 아키텍처와 우선순위 결정
- 완료 Gate와 사용자 검증 기준 관리
- Codex 작업 결과의 원격 GitHub 검증
- 다음 작업 범위 승인
- `CURRENT.md`, `PROJECT_STATE.json`, 로드맵 갱신

마스터 에이전트는 원칙적으로 긴 Codex 실행 프롬프트를 직접 작성하지 않는다. 단, 프롬프트 작성 에이전트의 기준 문서와 작업 지시를 관리한다.

### B. Codex 프롬프트 작성 에이전트 채팅

새 채팅이 담당한다.

- 마스터 에이전트가 승인한 다음 작업을 Codex 실행 프롬프트로 변환
- 현재 브랜치, 기준 커밋, 읽을 문서, 금지사항, 테스트, 산출물, 커밋·push 조건을 상세화
- Codex 결과를 사용자가 전달하면 누락된 보고 항목 확인
- 아키텍처 변경이 필요하면 직접 확정하지 않고 마스터 에이전트 검토를 요청

프롬프트 작성 에이전트는 다음을 독자적으로 변경하지 않는다.

- 전체 8단계 순서
- 현재 단계와 완료 Gate
- 채택된 HWPX 하이브리드 코어 구조
- 사용자 승인 없이 main merge
- 제품 범위 확대·축소

### C. Codex 실행 에이전트

- 지정된 브랜치에서 구현
- TDD, 진단, 산출물, 보고서 작성
- 원본 보호
- 새 브랜치 push
- 사용자 승인 없이 main merge 금지

### D. 사용자

- 제품 목표와 우선순위 승인
- 한글 2024 등 실제 화면 시각 검증
- 단계 완료와 다음 단계 진행 승인

## 2. 단일 진실 공급원

문서 우선순위:

1. `docs/gpt-communication/PROJECT_STATE.json` — 기계 판독용 현재 상태
2. `docs/gpt-communication/CURRENT.md` — 사람이 읽는 현재 상태와 다음 작업
3. `docs/gpt-communication/opinions/2026-07-02-army-claw-master-roadmap.md` — 전체 8단계와 Gate
4. `docs/gpt-communication/opinions/2026-07-02-hwpx-core-architecture-decision.md` — 현재 HWPX 코어 결정
5. 현재 작업 관련 최신 opinion 문서
6. 현재 작업 관련 최신 report 문서
7. GitHub 원격 브랜치·커밋·산출물 — 실제 구현 증거

충돌 시 최신 `PROJECT_STATE.json`과 `CURRENT.md`를 먼저 따르되, 로드맵 또는 아키텍처 변경은 마스터 에이전트 승인 기록이 있어야 한다.

## 3. 정보 전달 흐름

```text
마스터 에이전트
→ 현재 단계·다음 작업·Gate 확정
→ PROJECT_STATE.json / CURRENT.md 갱신

Codex 프롬프트 작성 에이전트
→ 위 문서를 읽고 상세 실행 프롬프트 작성

Codex
→ 구현·테스트·보고·push

사용자
→ 결과와 화면을 마스터 에이전트에 전달

마스터 에이전트
→ GitHub 원격 검증·시각 판정·다음 상태 결정
```

## 4. 프롬프트 작성 에이전트의 필수 시작 절차

모든 Codex 프롬프트 작성 전:

1. GitHub에서 `PROJECT_STATE.json` 확인
2. `CURRENT.md` 확인
3. 마스터 로드맵 확인
4. 현재 아키텍처 결정문 확인
5. 현재 기준 브랜치의 최신 커밋과 작업트리 전제 확인
6. 최신 관련 report·opinion 확인
7. 다음 작업의 입력·출력·완료 Gate 정리
8. 불명확하거나 충돌하는 항목은 프롬프트 작성 전 사용자에게 확인

## 5. Codex 프롬프트 필수 구성

- 전체 프로젝트 단계와 현재 세부 단계
- 이번 작업의 목적과 비목적
- 로컬 경로, 저장소, 기준 브랜치, 새 브랜치
- 반드시 읽을 문서
- 코드 수정 전 설계 판단
- 금지 명령과 원본 보호
- TDD 순서
- 구현 범위
- 회귀 테스트
- 사용자 검증용 산출물
- GPT 공유 보고서
- 커밋·push
- 사용자 확인 전 완료 선언 금지

## 6. 결과 회수 규칙

Codex 결과에는 최소 다음이 있어야 한다.

```text
branch
commit SHA
push 결과
변경 파일
테스트 결과
산출물 경로
diff/diagnostics 요약
미완료 항목
사용자 확인 항목
다음 재개 지점
```

Codex 결과를 마스터 에이전트가 검토할 때는 보고문만 신뢰하지 않고 GitHub 원격 커밋과 핵심 산출물을 확인한다.
